Upstart service items

Handles services managed by Upstart.

svc_upstart = {
    "gunicorn": {
        "running": True,  # default
    },
    "celery": {
        "running": False,
    },
}



Attribute reference

See also: The list of generic builtin item attributes

running

True if the service is expected to be running on the system; False if it should be stopped.

Canned actions

See also: Explanation of how canned actions work

reload

Reloads the service.

restart

Restarts the service.

stop

Stops the service.

stopstart

Stops and then starts the service. This is different from restart in that Upstart will pick up changes to the /etc/init/SERVICENAME.conf file, while restart will continue to use the version of that file that the service was originally started with. See http://askubuntu.com/a/238069.
